## Architecture expansion notes

**Expanding to 5 party members:**
`ALLY_SLOTS` in `BattleScene.ts` already has 5 entries (slots 3–4 reserved). `buildTurnOrder()` and the damage engine are party-size agnostic. Formation slot assignment is the only scene-level change.

**Adding multiple save slots:**
`SAVE_SLOT_KEY` in `save-types.ts` should become a function parameterised by slot number. `SAVE_VERSION` must be bumped if `SaveData` structure changes. Title scene needs a slot selection modal.

**Adding status effects:**
Add status fields to `Combatant` in `battle-types.ts`. Add application/tick logic in `battle-actions.ts`. Add visual indicators in `BattleScene.ts` status panel. Consider whether statuses persist to save data (if yes, `SAVE_VERSION` bump required).
